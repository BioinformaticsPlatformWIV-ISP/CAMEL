# CAMEL

CAMEL is the system that powers the bioinformatics workflows and stand-alone tools
maintained by the Sciensano Bioinformatics Platform.

Tools and workflows in CAMEL are designed with **traceability** and **reproducibility** as primary
goals, and therefore intentionally offer limited flexibility.

> **Note:** This is a large, complex codebase. You may run into bugs or compatibility
> issues on your system — please [open an issue](https://github.com/BioinformaticsPlatformWIV-ISP/CAMEL/issues)

----

## What is CAMEL?

- CAMEL runs the production bioinformatics workflows developed by the Sciensano
  Bioinformatics Platform.
- CAMEL is **not a sandbox** for running arbitrary tools or for interactively building
  custom workflows. While we welcome use of the framework, it is not a flexible,
  general-purpose bioinformatics suite.
- If you need extensive customization, we recommend a more flexible alternative such as
  [Bactopia](https://github.com/bactopia/bactopia).
- An overview of the published workflows is available in the
  [Citations](https://github.com/BioinformaticsPlatformWIV-ISP/CAMEL/wiki/Citations)
  section of the Wiki.

## Status & requirements

- ⚠️ Usage by external users is currently in **beta**.
- CAMEL requires substantial computational resources and storage.
- Not all tools and pipelines are guaranteed to run on every system.
